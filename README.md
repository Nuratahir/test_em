Здравствуйте. Тут хотел бы расписать в целом все свои действия по поводу решения тестового задания. Решил взять Django + PostgresSQL:
1. Первым делом сделал Git, создал репозиторий на GitHub
2. Устанавливаем менеджер пакетов - uv
3. В терминале инициализируем и добавляем django
4. После установление django, создал пакет-проект через команду _django-admin startproject config_ . Проверим, что все хорошо стартом проекта  _python manage.py runserver_
5. Далее создаем приложение с названием app - _python manage.py startapp app_
6. Зафиксировал в settings.py наше приложение INSTALLED_APPS = [..., 'app.apps.AppConfig',]
7. Далее хотел бы подключить базу данных в settings.py, PostgresSQL, поэтому в setting.DATABASES прописал все необходимое для подключения. Перед этим, все для подключения к БД я сделал Docker, через PyCharm подключился. Проверим подключение с помощью миграции. Все получилось, был конфликт с psycopg. Пришлось устанавливать psycopg[binary]. Для удобства, чтобы запускать одной командой, решил создать docker-compose.yml, чтобы сразу командой _docker-compose up -d_ запускать контейнер

9. Делаем кастомную модель пользователя. Где будет email - уникальный идентификатор для входа, ФИО и флаг is_active для удаления. Также для чистоты уберу прошлые данные из БД _docker-compose down -v_. Прописываю код в app/models.py
10. После кода, нужно указать в settings.py, чтобы Django знал, какую модель использовать. Теперь создаем и применяем миграции:
_uv run python manage.py makemigrations app_
_uv run python manage.py migrate_. Для проверки, пробуем создать суперпользователя: admin@ex.com, admin. 
11. Создаем модели для нашей кастомной системы прав. **ActionPermission хранит связку "ресурс: действие". Role — это группа таких прав. У пользователя может быть несколько ролей. Это и есть наша собственная логика, никак не связанная со встроенными**
12. Также делаем связь ManyToManyField (User: roles(Role)). Будет отдельная связующая таблица с ссылкой на user_id, role_id.
13. Сделал миграцию новых моделей. Для проверки, создал фикстуры с json данными, чтобы проверить, что отображается все и подгружается.
